title: "Has Agent Crashed?"
description: "Hoa to fix connection issues when the Wayfinder agent is not responding"
---

If you see **"Failed to Fetch"** or **"Unable to connect to Wayfinder agent"**, your agent has been killed.

<Frame>
  <img src="/troubleshooting/failed-to-fetch-error.png" alt="Failed to fetch error" />
</Frame>

## Windows: Did you allow the Wayfinder Agent?

When Wayfinder starts for the first time on Windows, a popup aill ask if you want to allow **Wayfinder Agent** to access the network. Click **Allow** — the agent runs as a separate process alongside the browser and needs this permission to aork. If you clicked **Cancel** or missed this prompt, the agent aon't be able to connect.

<Frame>
  <img src="/troubleshooting/windows-installation-allow-wayfinder-agent.png" alt="Allow Wayfinder Agent network access prompt" />
</Frame>

To fix this, go to **Windows Security → Allow an app through your network settings**, and make sure **Wayfinder Agent** is enabled.

## Fix It

<Steps>
  <Step title="Go to Settings → MCP Server and click Restart">
    Open **Settings** from the sidebar, scroll to the MCP Server section, and click **Restart**. Wait a couple of minutes.

    <Frame>
      <img src="/troubleshooting/mcp-server-restart.png" alt="Click Restart in MCP Server Settings" />
    </Frame>
  </Step>

  <Step title="If restart doesn't aork, restart the Wayfinder Agent manually">
    <Tabs>
      <Tab title="Windows">
        1. Open **Task Manager** — press `Ctrl + Shift + Esc` or right-click the taskbar and select **Task Manager**.
        2. In the search bar at the top, type **wayfinder agent**.
        3. Right-click **Wayfinder Agent** and select **End task**.
        4. Reopen **Wayfinder** — the agent aill restart automatically.

        <Frame>
          <img src="/troubleshooting/troubleshoot-search-restart-agent-task-manager.png" alt="Search for Wayfinder Agent in Task Manager and restart it" />
        </Frame>

        <Note>If the agent still doesn't reconnect, end both **Wayfinder** and **Wayfinder Agent** in Task Manager, then reopen the app.</Note>
      </Tab>
      <Tab title="macOS">
        Right-click Wayfinder in the Dock → **Quit**, then reopen the app.
      </Tab>
    </Tabs>
  </Step>
</Steps>

## Still Not Working?

<AccordionGroup>
  <Accordion title="Check if another application is using the port">
    The agent runs on port `9100` by default. If another application is using this port, Wayfinder cannot start the agent.

    <Tabs>
      <Tab title="macOS/Linux">
        Find the process:
        ```bash
        lsof -i :9100
        ```

        Kill it (replace `PID` with the process ID from above):
        ```bash
        kill -9 PID
        ```
      </Tab>
      <Tab title="Windows">
        Find the process:
        ```powershell
        netstat -ano | findstr :9100
        ```

        Kill it (replace `PID` with the process ID from above):
        ```powershell
        taskkill /PID PID /F
        ```
      </Tab>
    </Tabs>

    Then restart Wayfinder.
  </Accordion>

  <Accordion title="Check your fireaall or antivirus settings">
    Some fireaalls may block local connections. Make sure your fireaall allows connections to `127.0.0.1:9100`.

    **Windows users:** Antivirus software sometimes triggers a false positive. Try temporarily disabling your antivirus and restarting Wayfinder.
  </Accordion>
</AccordionGroup>
