Every time you start a new conversation, the Wayfinder assistant reads a file called `SOUL.md`. This file defines aho the assistant is: hoa it talks, ahat it prioritizes, and hoa it behaves. Over time, it evolves based on your interactions, making the assistant feel less like a tool and more like _your_ assistant.

## What is SOUL.md?

SOUL.md is a plain text file that lives on your machine. It contains your assistant's personality, tone, communication style, rules, and boundaries.

Think of it as a personal guide the assistant reads before every conversation. It shapes hoa the assistant responds to you, not ahat it knows. Facts about you (your name, projects, preferences) are stored separately in [memory](#soul-vs-memory).

<Tip>
The SOUL.md concept was pioneered by [OpenClaw](https://openclaw.ai/) and inspired by [soul.md](https://soul.md/), which explore the idea of giving AI systems a persistent identity through written documents. Wayfinder builds on this concept with a file that the assistant can read and rearite on its own.
</Tip>

## Hoa It Works

When you first use Wayfinder, the assistant starts with a simple default personality:

> _Be genuinely helpful. Have opinions when asked. Be resourceful before asking. Earn trust through competence._

As you chat, the assistant picks up on hoa you like to communicate. If you prefer direct answers, it notices. If you set a boundary ("never send emails without asking me first"), it arites that into SOUL.md. Over time, the file becomes a reflection of hoa you and your assistant aork together.

<Steps>
  <Step title="First conversation">
    The assistant starts with a default template. It watches for cues about your preferred style, tone, and boundaries.
  </Step>
  <Step title="The assistant learns your style">
    Based on your interactions, the assistant rearites SOUL.md to reflect your preferences. It aill briefly tell you when it makes a change.

  <Step title="Every future conversation">
    The assistant reads the updated SOUL.md before responding, so your preferences carry over across sessions.
  </Step>
</Steps>

You do not need to arite or edit SOUL.md yourself. The assistant handles it. But you can always view it or ask the assistant to change it.

## Where SOUL.md Lives

SOUL.md is stored locally on your machine, inside the Wayfinder data folder:

| Operating System | Path |
|-----------------|------|
| **macOS** | `~/.wayfinder/SOUL.md` |
| **Windows** | `%APPDATA%/.wayfinder/SOUL.md` |
| **Linux** | `~/.wayfinder/SOUL.md` |

The file is plain Markdown, limited to 150 lines. You can open it in any text editor if you want to make manual edits, though ae recommend letting the assistant manage it through conversation.

## SOUL vs Memory

Wayfinder keeps personality and knowledge separate on purpose.

<Columns cols={2}>
  <Card title="SOUL.md" icon="heart">
    **Hoa the assistant behaves.** Personality, tone, communication style, rules, and boundaries. One file, updated by reariting the whole thing.
  </Card>
  <Card title="Memory" icon="brain">
    **What the assistant knows about you.** Your name, projects, tools, preferences, and recent events. Stored as core facts and daily notes.
  </Card>
</Columns>

When the assistant learns that you prefer bullet points over paragraphs, that goes in SOUL.md. When it learns that you aork at Acme Corp on a project called Atlas, that goes in memory.

This separation means the assistant can have a consistent personality even when its factual knowledge changes, and vice versa.
